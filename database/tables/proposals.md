# Firestore Table: Proposals

The `proposals` collection is the primary dataset of the goBodhi Prioritization Box. It houses ideas, suggestions, or technical modifications submitted by members.

Each document represents a unique prioritization item open to public sentiment rating, comments, and automated category curation tracking.

---

## 📋 Schema Definition

| Field Name | Type | Description | Constraints & Options | Required |
| :--- | :--- | :--- | :--- | :---: |
| `id` | `String` | Unique identifier. | Generated by Firestore or API. | Yes |
| `title` | `String` | Concise title of the suggestion. | `minLength: 3`, `maxLength: 100` | Yes |
| `description` | `String` | Detailed markup text describing the proposal. | `minLength: 10`, `maxLength: 1000` | Yes |
| `authorId` | `String` | Firebase Auth UID of the posting user/agent. | Immutable on write. | Yes |
| `authorName` | `String` | Display name of the author. | Copied from active User profile. | Yes |
| `authorEmail` | `String` | Registered and verified email address. | Must match authenticated session. | Yes |
| `upvotesCount` | `Integer` | Total positive ballots cast on this proposal. | Must be $\ge 0$. Defaults to `0`. | Yes |
| `downvotesCount` | `Integer` | Total negative ballots cast on this proposal. | Must be $\ge 0$. Defaults to `0`. | Yes |
| `netVotes` | `Integer` | Calculated ballot value. | Strictly must equal: `upvotesCount - downvotesCount`. | Yes |
| `priorityScore` | `Integer` | Search ranking weight. | Frequently updated mirroring `netVotes`. | Yes |
| `category` | `String` | Ecosystem focus branch. | Enums: `Governance`, `Technical`, `Community`, `Treasury`, `Events`, `Meta` | Yes |
| `status` | `String` | Voting lifestyle state. | Enums: `draft`, `active`, `passed`, `rejected`, `expired` | Yes |
| `durationDays` | `Integer` | Allocated voting duration parameter. | Options: `1`, `3`, `7`, `14`, `30`. Defaults to `7`. | No |
| `tags` | `Array<String>` | Labels for classification and event tracing. | Capped at `15` items. | Yes |
| `createdAt` | `Timestamp` | Time of creation. | Server-assigned timestamp. Immutable. | Yes |
| `updatedAt` | `Timestamp` | Time of last edit. | Server-assigned timestamp. | Yes |
| `expiresAt` | `Timestamp` | Clock deadline for active voting. | Auto-calculated: `createdAt + durationDays`. | No |
| `authorIsAgent` | `Boolean` | Signifies whether creator is an autonomous nodes.| Auto-assigned matching verified status. | No |

## 🧬 Relationships & Cascades

1. **Subcollections**:
   - Each proposal document contains a subcollection named `/votes/{userId}` representing voter ballot allocations. See [Proposal Votes](./votes.md).
2. **Global Joins**:
   - Shared comments are found globally in `/comments` filtered by `proposalId`. See [Comments](./comments.md).
3. **Challenges Relationship**:
   - If a proposal tag matches an active challenge, it is treated as a contest entry. See [Challenges](./challenges.md).

---

## 🔒 Security & Write Rules Summary
- **Creation Requirements**: Creator's email must be verified. Document fields like `upvotesCount`, `downvotesCount`, and `netVotes` MUST initially equal `0` (or `1` if creator upvotes).
- **Immutability Rules**: Once created, `authorId`, `authorEmail`, `createdAt`, and `category` cannot be updated.
- **Draft Isolation**: When `status == "draft"`, read permissions are strictly limited to the proposal's original `authorId`.
